## Высокий приоритет

### 1. Расширенный поиск портов

**Диапазон портов**: `apf --port-range 8000-9000`  
  Позволяет сканировать сразу множество портов. Полезно для проверки dev-машин или локальных кластеров.

**Альтернативные движки поиска**: `--backend=ss|lsof|netstat|native`  
  На разных ОС доступны разные утилиты. Fallback-механизм обеспечит стабильность `apf` в любых окружениях.

---

### 2.️ Интерактивная и автоматическая реакция на конфликты

- [ ] **Интерактивное подтверждение Kill**  
  Пример: `Port 8080 is used by PID 1234 (node). Kill? [Y/n]`  
  Улучшает UX и снижает риск случайного завершения важных процессов.

- [ ] **Режим watch**: `apf --watch --port 8080 --kill`  
  Утилита следит за портом и автоматически завершает процессы, которые его занимают. Полезно для постоянных конфликтов (например, фоновый dev-сервер).

- [ ] **Умное завершение процесса**  
  - Поддержка выбора сигнала (`SIGTERM`, `SIGKILL`, `SIGINT`)  
  - Возможность убить вместе с дочерними  
  - Использование `sudo`, если недостаточно прав  
  - Возможность настроить правила: "если такой процесс занимает порт X — завершить всегда"

---

### 3. Логгирование и история

**Лог истории конфликтов**  
  Путь: `~/.apf/logs.json`  
  Формат:
  ```json
  [
    {"timestamp": "2025-07-22T20:22", "port": 3000, "pid": 1234, "name": "node", "action": "killed"}
  ]
```

Полезно для отладки и ретроспективы частых конфликтов.

## Средний приоритет

### 4. Интеграция с Docker

* [ ] **Показ информации о контейнере**
  Если порт занят Docker'ом — показать:
  `Port 5432 is used by Docker container: postgres-db`

* [ ] **docker-compose.yml / Procfile анализ**
  Сравнивать ожидаемые порты с занятыми. Показывать предупреждение, если нужный порт уже используется другим процессом.

---

### 5. Интеграции с DevOps / CI

* [ ] **Плагин для CI/CD**: `apf check --port 8080 --exit-if-used`
  Проверка перед деплоем или запуском тестов, что порт свободен.

* [ ] **Git hook / npm hook**
  Пример: при `git commit` или `npm start` проверять занятость порта. Удобно для локальных разработчиков.

---

## Низкий приоритет

### 6. Интерфейс и удобство

* [ ] **TUI (Text User Interface)**
  Позволяет в интерактивном режиме просматривать и управлять процессами:

    * Выбор стрелками
    * Нажатие `[K]` для Kill
    * Нажатие `[I]` для получения дополнительной информации о процессе

* [ ] **Автокомплит CLI команд**
  `apf completion bash|zsh|fish`
  Улучшает UX, особенно в терминалах с автодополнением.

---

### 7. Резервация портов

* [ ] **Планирование “забронированных” портов**
  Возможность указать, какие порты должны быть свободны под конкретные сервисы. Если порт занят — `apf` предупреждает.
